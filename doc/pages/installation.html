SeqMule Installation

Supported platform

x86-64 Linux


Prerequisites

make, gcc, g++, curses.h, R, unzip, automake, autoconf, git, gzip, tar, java


Install

Executables of SeqMule itself are basically some scripts in Perl, you can use them right after unpacking. However, external programs do need installation. SeqMule ignores what you have outside seqmule directory for sake of compatibility.


wget seqmule.usc.edu/seqmule.beta.tar.gz
tar zxvf seqmule.beta.tar.gz
cd seqmule
./Build status #show all commands and missing external programs
./Build freshinstall #install all external programs regardless of what you have

Alternatively, you can use:
./Build installexes #install all missing external programs

Due to copyright limitations, you have to download and install GATK (one of default variant callers) yourself. Use ./Build gatk to get instructions. If everything goes smooth, you are ready to go. Good luck!


NOTES

To install prerequisites, on RedHat, CentOS, Fedora, run

sudo yum install -y gcc gcc-c++ make ncurses-devel ncurses R unzip automake autoconf git-core gzip tar

To install prerequisites, on Ubuntu, run

sudo apt-get update sudo apt-get install -y build-essential gcc g++ ncurses-base ncurses-bin ncurses-term libncurses5 libncurses5-dev r-base unzip automake autoconf git gzip tar

To use seqmule without typing entire path, please modify your PATH environmental variable
In bash:

export PATH=$PATH:absolute_path_to_seqmule/bin

In tcsh:

setenv PATH $PATH:absolute_path_to_seqmule/bin

If you don't know which shell (bash/tcsh/csh etc.) you are using, type the following command to figure out


echo $0
